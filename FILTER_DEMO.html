🔍 Filtering System - Implementation Demo

✅ What's Been Implemented

  • Dynamic Filtering System - Filter offers by category, labels, price range, date range, and location
  • Combinable Filters - Apply multiple filter criteria simultaneously
  • Filter Subscriptions - Users can subscribe to filters and receive notifications for new matching offers
  • Smart Notifications - Only triggered for NEW offers created after subscription (not past ones)
  • Label Management - Managers can create and assign labels to offers
  • Filter Administration - Only admins can create and delete filter definitions
  • Visible Active Filters - UI clearly shows which filters are currently active
  • Role-Based Access Control - Admin, Manager, and User permissions properly configured

🎯 Filter Criteria

All criteria are optional and can be combined:

{
  "categories": ["Sale", "Rent", "Commercial"],
  "labelIds": [1, 2, 3],
  "minPrice": 10000,
  "maxPrice": 100000,
  "startDate": "2026-01-01T00:00:00Z",
  "endDate": "2026-12-31T23:59:59Z",
  "location": "Chișinău"
}

🔌 API Endpoints

👥 User Endpoints (Public & Authenticated)

GET /api/filters Public

List all active filters available for users

GET /api/filters/{id} Public

Get a specific filter by ID

POST /api/filters/apply Public

Apply filter criteria to get matching offers

{
  "categories": ["Sale"],
  "minPrice": 30000,
  "maxPrice": 80000,
  "location": "Centru"
}
POST /api/filters/{id}/subscribe Authenticated

Subscribe to a filter to receive notifications for new matching offers

DELETE /api/filters/{id}/subscribe Authenticated

Unsubscribe from a filter

GET /api/filters/subscriptions Authenticated

Get current user's filter subscriptions

👔 Manager Endpoints

GET /api/manager/labels Manager + Admin

List all labels

POST /api/manager/labels Manager + Admin

Create a new label

{
  "name": "Luxury",
  "description": "Premium high-end properties",
  "color": "#FFD700"
}
PUT /api/manager/labels/{id} Manager + Admin

Update a label

DELETE /api/manager/labels/{id} Manager + Admin

Delete a label

POST /api/manager/offers/{offerId}/labels/{labelId} Manager + Admin

Assign a label to an offer

DELETE /api/manager/offers/{offerId}/labels/{labelId} Manager + Admin

Remove a label from an offer

🔒 Admin Endpoints

GET /api/admin/filters Admin Only

List all filters (including inactive)

POST /api/admin/filters Admin Only

Create a new filter definition

  "description": "Apartments under 60,000 EUR in city center",
  "active": true,
  "criteria": {
    "categories": ["Sale"],
    "maxPrice": 60000,
    "location": "Centru"
  }
}
PUT /api/admin/filters/{id} Admin Only

Update a filter definition

DELETE /api/admin/filters/{id} Admin Only

Delete a filter

🔔 Notification Flow

  1. User subscribes to a filter - System records lastNotifiedAt = now()
  2. New offer is created - NotificationService.processNewOffer() is triggered
  3. System evaluates filters - Checks all active filters against the new offer
  4. Find matching subscriptions - For each matching filter, finds user subscriptions
  5. Smart notification logic - Only notifies if offer.createdAt > subscription.lastNotifiedAt
  6. Send notifications - Currently logged (extensible to email/Telegram/push)
  7. Update timestamp - Sets lastNotifiedAt = now() to prevent duplicates

🚀 How to Test

1. Start the Application

cd backend
mvn spring-boot:run

2. Access the Application

Open http://localhost:8080

The filter panel is visible on the main offers page!

3. Test with cURL

List Active Filters:

curl http://localhost:8080/api/filters

Apply Filters:

curl -X POST http://localhost:8080/api/filters/apply \
  -H "Content-Type: application/json" \
  -d '{
    "categories": ["Sale"],
    "minPrice": 30000,
    "maxPrice": 80000,
    "location": "Chișinău"
  }'

Create a Label (as Manager):

curl -X POST http://localhost:8080/api/manager/labels \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Premium",
    "description": "High-end properties",
    "color": "#FFD700"
  }'

Create a Filter (as Admin):

curl -X POST http://localhost:8080/api/admin/filters \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Budget Apartments",
    "description": "Affordable apartments under 50k",
    "active": true,
    "criteria": {
      "categories": ["Sale"],
      "maxPrice": 50000
    }
  }'
